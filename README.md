# FuNmiRBench

Benchmark functional miRNA target predictors against differential-expression tables.

## Requirements

- Python 3.10+
- `uv`

## Install

```bash
python -m pip install uv
git clone git@github.com:BioGeMT/FuNmiRBench.git
cd FuNmiRBench
uv sync
```

## Quick Start

The repo already ships with a runnable demo in `benchmark.yaml`.
It uses:

- 3 experiment datasets from `metadata/mirna_experiment_info.tsv`
- 2 demo predictors from `metadata/predictions_info.tsv`

Generate the local demo predictor TSVs, then run the benchmark:

```bash
uv run pipelines/standardized_predictors/mock/pipeline.py
uv run pipelines/standardized_predictors/cheating/pipeline.py
uv run funmirbench --config benchmark.yaml
```

Outputs are written to `results/`.

## Metadata

The two TSV files under `metadata/` are the source-of-truth registries for a run:

- `metadata/mirna_experiment_info.tsv`: one row per experiment dataset
- `metadata/predictions_info.tsv`: one row per predictor

They are not generated by the benchmark in this branch. They are curated input files that tell
the benchmark:

- which datasets exist
- which predictors exist
- which file paths to read
- what labels and attributes can be used for filtering

`data/` holds the actual TSV inputs. `metadata/` tells the benchmark what those files are.

## Config

All paths in the YAML are relative to the YAML file itself.

The default config looks like this:

```yaml
experiments_tsv: metadata/mirna_experiment_info.tsv
predictions_tsv: metadata/predictions_info.tsv

experiments:
  id: [GSE109725_OE_miR_204_5p, GSE118315_KO_miR_124_3p, GSE210778_OE_miR_375_3p]

predictors:
  tool_id: [predictor_1, predictor_2]

evaluation:
  fdr_threshold: 0.05
  abs_logfc_threshold: 1.0
  predictor_top_fraction: 0.10

out_dir: results/
```

What each section does:

- `experiments_tsv`: experiment registry
- `predictions_tsv`: predictor registry
- `experiments`: filters rows from the experiment registry
- `predictors`: filters rows from the predictor registry
- `evaluation`: ground-truth thresholds and predictor-correlation settings
- `out_dir`: where benchmark outputs are written

Filter behavior:

- AND across different keys
- OR within each list of values

`benchmark.yaml` already includes the other experiment and predictor columns as commented rows, so the normal workflow is just to edit or uncomment those filters.

## Predictor Files

The benchmark does not guess predictor file locations. It reads them from `metadata/predictions_info.tsv`.

For the demo:

- `predictor_1` points to `data/predictions/mock/mock_canonical.tsv`
- `predictor_2` points to `data/predictions/cheating/cheating_canonical.tsv`

Those files are generated locally by:

```bash
uv run pipelines/standardized_predictors/mock/pipeline.py
uv run pipelines/standardized_predictors/cheating/pipeline.py
```

They are not tracked by git.

## Outputs

After a run, `results/` contains:

- `joined/`: joined DE + predictor score tables
- `tables/`: APS, PR-AUC, AUROC, and Spearman summary tables
- `plots/<dataset_id>/`: per-dataset plots
- `reports/`: per-predictor text reports and predictor-correlation TSVs
- `summary.json`: run summary

When 2 or more predictors are selected, each dataset gets:

- one score-vs-logFC scatter per predictor
- one combined PR curve
- one combined ROC curve
- one algorithms-vs-genes heatmap
- one predictor-correlation heatmap

## Useful Commands

```bash
uv run funmirbench --config benchmark.yaml
uv run funmirbench-import-experiments --from-dir /path/to/tables
uv run funmirbench-validate-experiments --experiments-tsv metadata/mirna_experiment_info.tsv
uv run pytest
```

## Layout

```text
funmirbench/   package code and benchmark commands
pipelines/     GEO and standardized predictor pipelines
metadata/      experiment and predictor registries
data/          local benchmark inputs and generated predictor outputs
tests/         test suite
```
